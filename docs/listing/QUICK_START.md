# エラーハンドリング機能 - クイックスタート（1分で理解）

## 🎯 この機能は何？

**失敗した画像取得だけを自動で再処理できる機能です。**

---

## 📝 3ステップで使える

### 1️⃣ 画像を取得（いつも通り）
```
行を選択 → メニュー: eBay ツール → 📸 画像を取得（選択行）
```

失敗すると...
✅ 自動で「エラーログ」シートに記録される
✅ ポップアップで「エラーログに記録しました」と表示

---

### 2️⃣ 失敗した行を確認
```
メニュー: eBay ツール → 📋 エラーログを表示
```

ポップアップが出る：
```
未解決エラー: 5件

対象行番号: 15, 23, 31, 45, 52
```

---

### 3️⃣ 失敗した行だけ再処理
```
メニュー: eBay ツール → 🔄 失敗行を再処理
```

ポップアップが出る：
```
対象行数: 5件
✅ 成功: 4件
❌ 失敗: 1件

成功した行:
  15行目: 2枚取得
  23行目: 3枚取得
  31行目: 1枚取得
  45行目: 2枚取得

失敗した行:
  52行目: 画像が見つかりませんでした
```

**完了！**
- 成功した4行 → 画像URLが自動で追加される
- 失敗した1行 → 手動で確認・修正

---

## 💡 メリット

| 従来 | 新機能 |
|------|--------|
| 100行処理 → 5行失敗 → **全部やり直し** 😢 | 100行処理 → 5行失敗 → **5行だけやり直し** 😊 |
| 失敗した行が分からない | 自動で記録される |
| 手動で探す必要あり | ワンクリックで再処理 |

---

## 🎓 よくある使い方

### パターン1: 少量（1〜10行）
```
1. 行を選択 → 画像を取得
2. 失敗したら → エラーログを確認
3. 失敗行を再処理
```

### パターン2: 大量（100行以上）
```
1. clasp run 'runBatchImageFetch(2, 101)'
2. clasp run retryFailedRows
3. 完了！
```
