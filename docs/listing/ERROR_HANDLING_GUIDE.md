# エラーハンドリング機能 - 使い方ガイド

## 📚 この機能でできること

失敗した画像取得を**自動で記録**し、**失敗した分だけ再処理**できます。

### 従来の方法
```
100行処理 → 10行失敗
↓
全部やり直し（100行） 😢
無駄な時間と労力
```

### 新機能
```
100行処理 → 10行失敗
↓
失敗した10行だけ再処理 😊
効率的！時間節約！
```

---

## 🎯 3つの主要機能

### 1️⃣ **エラーログ自動記録**
画像取得に失敗すると、自動的に「エラーログ」シートに記録されます。

**記録される情報:**
- タイムスタンプ（いつ失敗したか）
- 行番号（どの行か）
- エラー種別（ネットワークエラー、画像なし、など）
- エラーメッセージ（詳細）
- 仕入元URL
- 商品ID
- 状態（未処理/再処理中/解決済み）

**色分け:**
- 🔴 赤：未処理（まだ解決していない）
- 🟡 黄：再処理中（今処理している）
- 🟢 緑：解決済み（成功した）

---

### 2️⃣ **失敗行の自動検出**
「エラーログ」から未解決の行を自動で見つけます。

---

### 3️⃣ **失敗行の一括再処理**
失敗した行だけを自動で再処理し、結果をポップアップで表示します。

---

## 📖 使い方（4ステップ）

### ステップ1: 画像を一括取得

**方法1: clasp runで実行**
```bash
clasp run 'runBatchImageFetch(2, 101)'
```
↓ 結果
```
処理: 2〜101行目（100行）
成功: 90行
失敗: 10行 ← 自動的にエラーログに記録される
```

**方法2: スプレッドシートのメニューから**
```
各行を選択 → eBay ツール → 📸 画像を取得（選択行）
```

---

### ステップ2: エラーログを確認

**方法1: スプレッドシートで確認**
```
1. スプレッドシートの下部タブで「エラーログ」シートをクリック
2. 失敗した行の詳細を確認
```

**方法2: ポップアップで確認**
```
スプレッドシートのメニュー:
eBay ツール → 📋 エラーログを表示
```

↓ ポップアップが表示される

### ステップ3: 失敗行を再処理

**方法1: clasp runで実行**
```bash
clasp run retryFailedRows
```

**方法2: スプレッドシートのメニューから（推奨）**
```
スプレッドシートのメニュー:
eBay ツール → 🔄 失敗行を再処理
```

↓ 確認ダイアログが表示される
```
━━━━━━━━━━━━━━━━━━━━
失敗行の再処理

10件の失敗した行を再処理しますか？

対象行: 15, 23, 31, 45, 52, 67, 72, 81, 89, 95

[はい]  [いいえ]
━━━━━━━━━━━━━━━━━━━━
```

「はい」をクリック

↓ 処理中...（進捗表示）
```
画面右下に通知:
「15行目を再処理中... (1/10)」
「23行目を再処理中... (2/10)」
...
```

3. 📊 開発用シート → 画像URLが追加されている

## 💡 便利な機能

### エラーログをクリア
```
スプレッドシートのメニュー:
eBay ツール → 🗑️ エラーログをクリア
```
すべてのエラーログを削除します（取り消せないので注意）。

## 🎓 実例：100商品の画像を取得する場合

### シナリオ
メルカリから100商品の画像を取得したい。
でも、いくつかの商品は画像が削除されているかもしれない。

### 従来の方法
```
9:00  - 手動で1行ずつ処理開始
10:40 - 100行完了
      → でも10行が失敗していることに気づかない 😢

翌日 - 失敗していることに気づく
     - どの行が失敗したか分からない
     - 全部やり直し（1時間40分）
```

### 新機能を使った場合
```
9:00  - バッチ処理開始
        clasp run 'runBatchImageFetch(2, 101)'

10:00 - 処理完了
        → 90行成功、10行失敗
        → 失敗した10行は自動でエラーログに記録 ✅

10:01 - エラーログを確認
        eBay ツール → 📋 エラーログを表示
        → 失敗理由を確認
        → 「ネットワークエラー」が多い

10:05 - 失敗した10行だけ再処理
        eBay ツール → 🔄 失敗行を再処理
        → 8行成功、2行失敗

10:10 - 残り2行を手動確認
        → URLが間違っていた → 修正 → 再処理
        → すべて完了！ 🎉

合計時間: 1時間10分（従来は3時間20分）
節約時間: 2時間10分！
```

## 📊 エラー種別の説明

| エラー種別 | 意味 | 対処法 |
|-----------|------|--------|
| ネットワークエラー | メルカリサーバーにアクセスできない | 時間をおいて再処理 |
| 画像なし | 商品に画像が登録されていない | URLを確認、商品ページを確認 |
| タイムアウト | 処理時間が長すぎる | 再処理で解決することが多い |
| URL未入力 | 仕入元URLが空 | URLを入力してから再処理 |
| その他エラー | 予期しないエラー | エラーメッセージを確認 |

---

## ⚠️ 注意事項

1. **エラーログは自動で蓄積されます**
   - 定期的に「エラーログをクリア」で整理推奨

2. **同じ行で複数回失敗すると、すべて記録されます**
   - エラーログに同じ行番号が複数行表示される場合あり

3. **「解決済み」になった行も記録は残ります**
   - 履歴として確認できる

4. **clasp runとメニュー実行の違い**
   - clasp run: スプレッドシートを開かなくても実行可能
   - メニュー: ポップアップで結果が見やすい（推奨）

## 📞 トラブルシューティング

### Q: エラーログシートが見つかりません
A: 初回エラー発生時に自動作成されます。画像取得を一度実行してください。

### Q: 再処理しても同じエラーが出ます
A: エラーメッセージを確認し、根本原因（URL間違い、商品削除など）を解決してください。

### Q: ポップアップが表示されません
A: スプレッドシートのメニューから実行してください。clasp runではポップアップは表示されません。
