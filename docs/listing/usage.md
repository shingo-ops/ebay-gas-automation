## 基本操作

### カスタムメニュー

スプレッドシートを開くと、メニューバーに「eBay API」メニューが追加されます：

```
eBay API
├── 🔐 認証
├── ────────
├── 🔍 商品検索
├── 📦 在庫同期
├── 📋 注文取得
├── ────────
├── 📊 ダッシュボード作成
├── ────────
├── ⚙️ 設定
└── ℹ️ バージョン情報
```

### 初回起動時の認証

1. 「eBay API」→「認証」を選択
2. 自動的に eBay API のアクセストークンを取得
3. 成功メッセージが表示されれば準備完了
4. トークンの有効期限は 2 時間（自動更新されます）

## 商品検索機能

### 基本的な商品検索

1. **メニューから実行**
   - 「eBay API」→「商品検索」を選択
   - 検索ダイアログが表示されます

2. **検索キーワード入力**
   - 検索したい商品のキーワードを入力（例: `iPhone 14`）
   - オプション設定:
     - 取得件数（デフォルト: 50件）
     - ソート順（価格昇順/降順、新着順など）

3. **検索実行**
   - 「検索」ボタンをクリック
   - 検索結果が「商品マスタ」シートに表示されます

### 商品マスタシートの構成

検索結果は以下の列で表示されます：

| 列 | 項目 | 説明 |
|---|---|---|
| A | Item ID | eBay の商品ID |
| B | タイトル | 商品名 |
| C | 価格 | 販売価格 |
| D | 通貨 | 通貨コード（USD, JPY など） |
| E | カテゴリ | 商品カテゴリ |
| F | コンディション | 新品/中古など |
| G | 在庫状況 | 販売中/終了 |
| H | URL | 商品ページURL |
| I | 画像URL | 商品画像のURL |
| J | 更新日時 | データ取得日時 |

### 高度な検索機能

#### スクリプトエディタから実行

Apps Script エディタで `Products.gs` を開き、以下の関数を使用：

```javascript
// キーワード検索
searchAndWriteProducts('laptop', {
  limit: 100,        // 取得件数
  sort: 'price',     // ソート順（price, newlyListed など）
  filter: 'price:[100..500]'  // フィルター（価格範囲など）
});

// カテゴリ別検索
searchProductsByCategory('58058', {  // カテゴリID
  limit: 50
});
```

## 在庫管理機能

### 在庫シートの準備

1. **シートの作成**
   - 新しいシート「在庫」を作成（自動作成される場合もあります）

2. **在庫データの構成**

| 列 | 項目 | 説明 |
|---|---|---|
| A | SKU | 商品の SKU（在庫管理コード） |
| B | 商品名 | 商品名 |
| C | 在庫数 | 現在の在庫数 |
| D | ロケーション | 保管場所（オプション） |
| E | ステータス | 商品の状態 |
| F | 更新日時 | 最終更新日時 |

### eBay から在庫を同期

1. **在庫同期の実行**
   - 「eBay API」→「在庫同期」を選択
   - eBay の在庫情報をスプレッドシートに取得

2. **同期の仕組み**
   - 在庫シートの SKU 列に基づいて eBay から最新情報を取得
   - 既存の行を更新、新規の SKU は追加

3. **注意事項**
   - 大量の SKU がある場合は時間がかかります
   - API レート制限により、処理が遅延する場合があります

### スプレッドシートから eBay へ在庫を更新

1. **在庫数の編集**
   - 在庫シートの「在庫数」列（C列）を編集

2. **eBay への反映**
   - Apps Script エディタで `Inventory.gs` を開く
   - 関数 `updateInventoryToEbay` を実行
   - 確認ダイアログで「はい」を選択

3. **更新結果の確認**
   - 成功した件数とエラー件数が表示されます
   - エラーがある場合は「ログ」シートで詳細を確認

### 在庫アラート機能

在庫が少なくなった商品を自動で通知：

```javascript
// Apps Script エディタで実行
checkInventoryAlerts(10);  // 閾値: 10個
```

- 在庫数が指定の閾値以下の商品をリストアップ
- アラートダイアログで表示

### 在庫レポート

在庫状況の統計レポートを生成：

```javascript
// Apps Script エディタで実行
generateInventoryReport();
```

レポート内容：
- 総商品数
- 総在庫数
- 平均在庫数
- 低在庫商品数

## 注文管理機能

### 注文情報の取得

1. **メニューから実行**
   - 「eBay API」→「注文取得」を選択
   - 最新の注文情報を取得

2. **注文管理シートの構成**

| 列 | 項目 | 説明 |
|---|---|---|
| A | 注文ID | eBay の注文ID |
| B | 購入者名 | 購入者の名前 |
| C | 注文日時 | 注文が確定した日時 |
| D | 商品名 | 購入された商品 |
| E | 金額 | 合計金額 |
| F | ステータス | 注文ステータス |
| G | 配送先 | 配送先住所 |
| H | 追跡番号 | 配送の追跡番号 |

### 注文フィルタリング

特定の条件で注文を取得：

```javascript
// Apps Script エディタで実行
const orders = getOrders({
  filter: 'orderfulfillmentstatus:{IN_PROGRESS}',  // フルフィルメント中
  limit: 100
});
```

フィルター例：
- `orderfulfillmentstatus:{NOT_STARTED}`: 未処理
- `orderfulfillmentstatus:{IN_PROGRESS}`: 処理中
- `orderfulfillmentstatus:{FULFILLED}`: 完了

## ダッシュボード機能

### ダッシュボードの作成

1. **メニューから実行**
   - 「eBay API」→「ダッシュボード作成」を選択

2. **表示内容**
   - 総商品数
   - 総在庫数
   - 総注文数
   - 今月の売上
   - 前月の売上
   - 最終更新日時

3. **自動更新**
   - ダッシュボードは他のシートのデータを参照しているため、元データが更新されると自動的に反映されます

### カスタマイズ

ダッシュボードは自由にカスタマイズできます：

- グラフの追加
- カスタム集計式の追加
- 条件付き書式の設定

## 自動化とトリガー

### 定期的な自動実行

Apps Script のトリガー機能を使用して、定期的にデータを更新できます。

1. **トリガーの設定**
   - Apps Script エディタで「トリガー」（時計アイコン）をクリック
   - 「トリガーを追加」をクリック

2. **在庫同期の自動化例**
   - 実行する関数: `syncInventory`
   - イベントのソース: `時間主導型`
   - 時間ベースのトリガー: `日付ベースのタイマー`
   - 時刻: `午前 9 時〜10 時`

3. **注文取得の自動化例**
   - 実行する関数: `fetchOrders`
   - イベントのソース: `時間主導型`
   - 時間ベースのトリガー: `時間ベースのタイマー`
   - 間隔: `1 時間おき`

### トリガーの注意事項

- トークンの有効期限は 2 時間のため、長時間の自動実行には refresh token の実装が必要です
- API レート制限に注意してください
- エラーが発生した場合はメール通知を設定できます

## Tips と活用例

### 1. 大量データの処理

大量の商品や注文を処理する場合：

```javascript
// バッチ処理の例
const itemIds = [...]; // 大量の Item ID
const batches = chunkArray(itemIds, 50);  // 50件ずつに分割

batches.forEach((batch, index) => {
  Logger.log('バッチ ' + (index + 1) + '/' + batches.length);
  getMultipleProductDetails(batch);
  Utilities.sleep(5000);  // 5秒待機（レート制限対策）
});
```

### 2. データのエクスポート

スプレッドシートのデータを CSV でエクスポート：

```javascript
exportProductsToCSV();
```

### 3. カスタム関数の追加

独自の処理を追加する場合は、新しい `.gs` ファイルを作成：

```javascript
// CustomFunctions.gs
function myCustomFunction() {
  // あなたの処理
}
```

### 4. エラーログの確認

エラーが発生した場合は「ログ」シートで詳細を確認：

- タイムスタンプ
- ログレベル（INFO, WARNING, ERROR）
- メッセージ
- 詳細情報

### 5. パフォーマンスの最適化

- 必要なデータのみを取得（limit パラメータの活用）
- キャッシュの利用（CacheService）
- バッチ処理の活用

## よくある質問

### Q: トークンの有効期限が切れた場合は？

A: 自動的に再取得されますが、手動で「eBay API」→「認証」を実行することもできます。

### Q: Sandbox と Production の切り替え方法は？

A: `Config.gs` の `EBAY_ENVIRONMENT` を変更します：
```javascript
EBAY_ENVIRONMENT: 'PRODUCTION'  // 本番環境
```

### Q: API レート制限に達した場合は？

A: `Config.gs` の `RATE_LIMIT_DELAY` を増やすか、処理を分散させてください。

### Q: 複数のマーケットプレイスに対応できますか？

A: はい。`EBAY_MARKETPLACE_ID` を変更することで、異なるマーケットプレイス（US, UK, DE など）に対応できます。
