### Phase 1b: Item Specifics抽出機能

#### 機能概要
- 競合eBay URLから商品詳細を自動取得
- Item Specifics（Brand, Model, Color等）の抽出
- スプレッドシートへの自動入力

#### 成果物
| ファイル名 | 説明 | 行数（推定） |
|-----------|------|-------------|
| `ItemSpecificsExtractor.gs` | Item Specifics抽出 | 250行 |
| `UrlParser.gs` | URL解析・Item ID抽出 | 150行 |

#### 実装内容
1. **URL解析**
   - eBay URL パターンマッチング
   - Item ID 抽出
   - 短縮URL展開（オプション）

2. **商品データ取得**
   - Browse API（getItem）の利用
   - Item Specifics の全取得
   - 商品タイトル・説明・画像URL

3. **スプレッドシート統合**
   - 「Item Specifics抽出」シート作成
   - URL貼り付け → 自動取得
   - 一括取得機能

#### 工数内訳
| タスク | 時間 | 説明 |
|--------|------|------|
| URL解析実装 | 3h | パターンマッチング |
| API統合 | 3h | getItem の実装 |
| データマッピング | 4h | Specifics → シート |
| テスト・デバッグ | 2h | 動作確認・修正 |
| **小計** | **12h** | |

### Phase 1d: 出品機能（Best Offer含む）

#### 機能概要
- スプレッドシートから eBay へ出品
- 3ステップ出品（在庫アイテム → オファー → 公開）
- Best Offer 設定対応
- 単一・一括出品
- ステータス管理

#### 成果物
| ファイル名 | 説明 | 行数（推定） |
|-----------|------|-------------|
| `Listing.gs` | 出品機能メイン | 400行 |
| `ValidationRules.gs` | バリデーション | 300行 |
| `OfferManager.gs` | Offer管理 | 250行 |
| `StatusManager.gs` | ステータス管理 | 150行 |

#### 実装内容
1. **在庫アイテム作成**
   - Inventory API 統合
   - PUT /inventory_item/{sku}
   - 商品データマッピング

2. **オファー作成**
   - POST /offer
   - 価格・在庫設定
   - **Best Offer 設定**
     - bestOfferEnabled
     - autoAcceptPrice
     - autoDeclinePrice
   - ビジネスポリシー設定

3. **リスティング公開**
   - POST /offer/{offerId}/publish
   - Listing ID 取得
   - ステータス更新

4. **バリデーション**
   - 必須項目チェック
   - データ型検証
   - SKU重複チェック
   - カテゴリID検証
   - **Best Offer 論理チェック**
     - 価格の順序関係
     - カテゴリサポート確認

5. **一括出品**
   - 選択行の一括処理
   - バッチ処理（50件ずつ）
   - プログレスバー表示
   - エラーハンドリング

6. **ステータス管理**
   - 未出品 → 検証中 → 出品中
   - エラー状態の管理
   - 色分け表示

7. **スプレッドシート統合**
   - 「出品管理」シート設計
   - Offer ID / Listing ID 記録
   - エラーメッセージ記録
   - 出品日時記録

## ⚠️ リスクと前提条件

### 前提条件

1. **eBay Developer Account が利用可能**
   - Client ID / Secret が取得済み
   - OAuth 認証が設定済み
   - Production 環境へのアクセス権限あり

2. **Google アカウント**
   - Google Apps Script が利用可能
   - スプレッドシートへのアクセス権限

3. **既存コードベース**
   - `EbayAPI.gs` が実装済み（searchItems, getItem等）
   - `Utils.gs` が実装済み
   - `Config.gs` が設定済み

4. **テストデータ**
   - テスト用の商品データが準備可能
   - Sandbox 環境でのテストが可能
