## 初期設定手順

### Step 1: ツール設定シートの準備

スプレッドシートに「ツール設定」シートを作成し、以下の情報を入力：

| 設定項目 | 説明 | 例 |
|---------|------|-----|
| **App ID** | eBay App ID | yourappid123 |
| **Cert ID** | eBay Cert ID | ************ |
| **Dev ID** | eBay Dev ID | yourdevid456 |
| **Sandbox** | サンドボックス使用 | FALSE |
| **画像フォルダ** | Googleドライブフォルダ URL | https://drive.google.com/drive/folders/xxx |
| **出品シート** | 出品シートスプレッドシート URL | https://docs.google.com/spreadsheets/d/xxx |
| **カテゴリマスタ** | カテゴリマスタスプレッドシート URL | https://docs.google.com/spreadsheets/d/xxx |

---

### Step 2: ワンクリック初期設定（初回のみ）

#### 方法1: メニューから実行（推奨）

1. スプレッドシートを開く
2. メニューバー「初期設定」をクリック
3. 「🎉 ワンクリック初期設定（推奨）」を選択
4. ステップ1: 権限承認画面で「続行」をクリック
5. Googleアカウントを選択
6. 「詳細」→「安全でないページに移動」をクリック
7. 必要な権限を確認して「許可」をクリック
8. ステップ2: onOpenトリガーが自動設定される
9. ステップ3: 設定検証が自動実行される
10. 完了メッセージが表示されたら、スプレッドシートをリロード（F5キー）

**このボタン1つで、権限承認・トリガー設定・検証がすべて完了！**

#### 方法2: 画像ボタンから実行

画像ボタンの設置方法は `IMAGE_BUTTON_SETUP.md` を参照

---

### Step 3: 編集時トリガーの設定（オプション）

リサーチシートのURL欄を編集したときに自動的にカテゴリ情報を取得：

1. メニュー「初期設定」→「個別設定（上級者向け）」→「トリガー管理」→「編集時トリガー有効化」
2. 完了メッセージが表示される
3. リサーチシートのB8セル（Item URL）を編集してテスト

**注意**: Step 2のワンクリック初期設定で、onOpenトリガーは既に設定済みです。

## 権限承認

### 承認される権限

初回の権限承認で以下の権限が要求されます：

| 権限 | 用途 |
|------|------|
| **スプレッドシートの表示と編集** | データの読み書き |
| **Googleドライブのファイル管理** | 画像の保存 |
| **外部サービスへの接続** | eBay APIへのリクエスト |

### セキュリティについて

- ✅ 承認した権限は、このスプレッドシートでのみ使用されます
- ✅ 権限は後からGoogleアカウント設定で取り消すことができます
- ⚠️ 「このアプリは Google で確認されていません」と表示されるのは、自作スクリプトのため正常な挙動です

## トリガー設定

### 設定可能なトリガー

| トリガー種類 | 関数名 | タイミング | 用途 |
|------------|--------|----------|------|
| **onOpen** | onOpen | スプレッドシートを開いたとき | メニュー表示 |

### トリガー管理コマンド

| メニュー項目 | 機能 |
|------------|------|
| **onOpenトリガー設定** | onOpenトリガーを新規作成 |
| **編集時トリガー有効化** | onEditトリガーを有効化 |
| **編集時トリガー無効化** | onEditトリガーを無効化 |
| **トリガー一覧表示** | 設定中のトリガーを表示 |
| **⚠️ すべてのトリガー削除** | 全トリガーを削除（リセット用） |

## 動作確認

### 1. 設定検証

1. メニュー「初期設定」→「設定を検証」
2. すべての設定項目にチェックマークが表示されることを確認

### 2. 画像ダウンロード機能

1. リサーチシートのG11セル（画像URL）に画像URLを入力
2. メニュー「eBay API」→「出品」
3. 「画像をダウンロード中...」メッセージが表示される
4. 出品シートの「画像1」列にGoogleドライブのURLが出力される

### 3. Item Specifics取得

1. リサーチシートのB8セル（Item URL）にeBay商品URLを入力
2. D8セル（Spec URL）にスペック取得用URLを入力
3. メニュー「eBay API」→「出品」
4. 出品シートにItem Specificsが色分けされて出力される
   - 🔴 赤色 = 必須項目
   - 🔵 青色 = 推奨項目
   - ⚫ グレー = その他項目

## トラブルシューティング

### Q1: 「初期設定」ボタンをクリックしても反応しない

**原因**: スクリプトが正しく割り当てられていない

**対処法**:
1. 画像/図形を右クリック
2. 「スクリプトを割り当て」を確認
3. 関数名が `completeInitialSetup` になっているか確認
4. スペルミスがあれば修正

**代替方法**:
- メニュー「初期設定」→「🎉 ワンクリック初期設定（推奨）」から実行

---

### Q2: トリガーが動作しない / メニューが表示されない

**原因**: トリガーが正しく設定されていない、または削除された

**対処法**:
1. メニュー「初期設定」→「個別設定（上級者向け）」→「トリガー管理」→「トリガー一覧表示」
2. 必要なトリガーが表示されているか確認
3. 表示されていない場合は、「🎉 ワンクリック初期設定（推奨）」を再実行

---

### Q3: 画像がダウンロードされない

**原因1**: 画像フォルダが設定されていない

**対処法**:
1. ツール設定シートの「画像フォルダ」を確認
2. GoogleドライブフォルダのURLが正しく入力されているか確認

**原因2**: 権限が承認されていない

**対処法**:
1. メニュー「初期設定」→「🎉 ワンクリック初期設定（推奨）」を再実行

---

### Q4: 「セル M10 に入力したデータは、このセルのデータ入力規則に違反しています」

**原因**: ヘッダーマッピング機能により解決済み（最新版では発生しない）

**対処法**:
1. 最新版のスクリプトがデプロイされているか確認
2. `clasp push` でデプロイ
3. スプレッドシートをリロード

---

### Q5: 設定検証でエラーが表示される

**対処法**:
1. メニュー「初期設定」→「設定を表示」でエラー内容を確認
2. ツール設定シートの該当項目を修正
3. 再度「設定を検証」を実行

## サポート

### Setup.gs に集約された関数一覧

#### 1. 権限承認セクション
- `completeInitialSetup()` - ⭐ ワンクリック初期設定（推奨）
- `authorizePermissions()` - 全権限承認実行（個別設定用）
- `showAuthorizationGuide()` - 権限承認ガイド表示

#### 2. トリガー管理セクション
- `setupOnOpenTrigger()` - onOpenトリガー設定
- `enableEditTrigger()` - 編集時トリガー有効化
- `disableEditTrigger()` - 編集時トリガー無効化
- `removeAllTriggers()` - 全トリガー削除
- `showTriggers()` - トリガー一覧表示

#### 3. 設定管理セクション
- `initialSetup()` - 初期セットアップ実行
- `showConfig()` - 設定表示
- `checkConfig()` - 設定検証

#### 4. イベントハンドラセクション
- `onEdit(e)` - 編集時実行関数

### 画像ボタン推奨構成

**最小2ボタンで完璧：**
1. `completeInitialSetup` - 初期設定（初回のみ）
2. `transferListingData` - 出品（毎回使用）

## 付録: デプロイ手順（管理者向け）

### 本番デプロイ

```bash
# 開発用ファイルを除外してデプロイ
clasp push

# .claspignore により以下が自動除外される:
# - 開発・デバッグ用ファイル（17ファイル）
# - ドキュメント（*.md）
```

### 全ファイルデプロイ（開発時）

```bash
# .claspignore を一時的に無効化
mv .claspignore .claspignore.bak
clasp push
mv .claspignore.bak .claspignore
```
