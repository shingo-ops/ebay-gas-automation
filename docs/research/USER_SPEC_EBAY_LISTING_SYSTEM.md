## システム概要

### このツールは何をするもの？

eBayで商品をリサーチして出品するまでの作業を自動化するツールです。

**手作業でやると**:
1. eBayで商品情報をコピペ
2. 仕入先のサイトから画像を保存
3. 画像をGoogleドライブにアップロード
4. 出品シートに情報を転記
5. 利益計算

**このツールを使うと**:
1. eBay URLと仕入元URLを入力
2. 出品ボタンを押す
3. **自動で全部完了！**

---

## 主な機能

### 1. 商品情報の自動取得
- eBay APIから商品タイトル、カテゴリ、価格などを自動取得
- Item Specifics（商品仕様）を最大30件まで自動充填

### 2. 画像の自動ダウンロード
- メルカリ・ヤフオクの商品ページから画像を自動抽出
- 最大20枚まで対応
- Googleドライブに自動保存
- 出品シートに画像URLを自動出力

### 3. ポリシー別出品（3種類）
- **Expedited（速達）**: 早く届ける発送方法
- **Standard（通常）**: 通常の発送方法
- **書状**: 安価な発送方法

それぞれのボタンを押すと、その発送方法での利益計算結果が出品シートに反映されます。

### 4. 仕入元の自動判定
ツール設定シートに登録したサイト（Amazon、メルカリなど）を自動で判定して、仕入元列に名前を出力します。

### 5. 複数人同時作業対応
複数人が同時に出品ボタンを押しても、データが上書きされない仕組みを実装。
**SKU（在庫管理番号）**を先に出力して行を予約します。

## 使い方

### 事前準備

#### 1. ツール設定シートの設定

| 項目 | 内容 | 例 |
|------|------|-----|
| App ID | eBay APIのアプリケーションID | 指定のID |
| Cert ID | eBay APIの証明書ID | 指定のID |
| Dev ID | eBay APIの開発者ID | 指定のID |
| 画像フォルダ | 画像を保存するGoogleドライブフォルダのURL | https://drive.google.com/drive/folders/xxxxx |
| 出品シート | 出品データを出力するスプレッドシートのURL | https://docs.google.com/spreadsheets/d/xxxxx |
| カテゴリマスタ | カテゴリ情報を管理するスプレッドシートのURL | https://docs.google.com/spreadsheets/d/xxxxx |

#### 2. 仕入元マッピングの設定（ツール設定シート）

| 仕入元 | URL |
|--------|-----|
| Amazon | https://www.amazon.co.jp/ |
| メルカリ | https://jp.mercari.com/ |
| ヤフオク | https://auctions.yahoo.co.jp/ |
| ラクマ | https://fril.jp/ |
| Yahoo!フリマ | https://paypayfleamarket.yahoo.co.jp/ |
| Yahoo!ショッピング | https://shopping.yahoo.co.jp/ |
| 楽天市場 | https://www.rakuten.co.jp/ |

### 基本的な使い方

#### ステップ1: リサーチシートに情報を入力

**トップ情報（B2～C2）**
- リサーチ方法: 「利益」「ベストオファー」など
- 担当者名: 自分の名前

**価格情報（E4～H5）**
- 仕入元URL①: メルカリやヤフオクなどの商品URL
- 画像URL: 画像を取得する商品ページURL（仕入元URL①と同じことが多い）
- メモ: 自由記入

**商品リスト（E7～H12）**
- Item URL: eBayの商品URL
- 状態: New, Used, For parts or not workingなど

**メイン情報（K4～L11）**
- 仕入値(¥): 仕入れ価格（円）
- 売値($): 販売価格（ドル）
- Best offer: ベストオファーの可否
- 実重量(g): 商品の重さ
- 奥行き(cm), 幅(cm), 高さ(cm): 商品サイズ

**ポリシーセクション（E13～H16）**
- 13行目: ヘッダー（ポリシー、発送方法、利益額、利益率）
- 14行目: Expedited（速達）の計算結果
- 15行目: Standard（通常）の計算結果
- 16行目: 書状の計算結果

#### ステップ2: 出品ボタンを押す

リサーチシートに3つのボタンがあります：

1. **出品（Expedited）**: 速達配送での出品
2. **出品（Standard）**: 通常配送での出品
3. **出品（書状）**: 書状配送での出品

希望する発送方法のボタンをクリックします。

#### ステップ3: 確認ダイアログ

「{ポリシー名} shippingで出品しますか？」というダイアログが表示されます。

- **OK**: 出品を実行
- **キャンセル**: 中止

#### ステップ4: 自動処理の開始

**処理の流れ**（すべて自動）:

1. **SKUを生成して行を予約**
   - SKU形式: `リサーチ方法/担当者/利益額/利益率/タイムスタンプ`
   - 例: `利益/田中/1500/25/20260328143052`

2. **eBay APIから商品情報を取得**
   - タイトル、カテゴリ、Brand、UPC、EAN、MPN
   - Item Specifics（最大30件）

3. **商品ページから画像を取得**
   - メルカリ・ヤフオクから最大20枚の画像を抽出
   - Googleドライブに保存
   - 出品シートに画像URLを出力

4. **出品シートに転記**
   - 商品情報、価格情報、Item Specificsなど全て自動で出力
   - 個数: 自動で「1」を設定
   - 仕入元: URL①②③の順にチェックして自動判定

#### ステップ5: 完了

「出品データを転記しました」というメッセージが表示されたら完了です。

## 各シートの役割

### 1. ツール設定シート
- **目的**: APIキーやフォルダURLなどの設定を管理
- **編集頻度**: 初期設定のみ（普段は触らない）

### 2. リサーチシート
- **目的**: 商品リサーチ情報を入力
- **編集頻度**: 毎回の出品時

### 3. 出品シート
- **目的**: 出品データを管理（自動で書き込まれる）
- **編集頻度**: 必要に応じて手動修正

### 4. カテゴリマスタシート
- **目的**: eBayカテゴリごとの必須・推奨Item Specificsを管理
- **編集頻度**: ほぼなし

## ポリシー別出品機能

### SKU（在庫管理番号）とは？

SKUは商品を一意に識別するための番号です。

**フォーマット**:
```
リサーチ方法/担当者/利益額/利益率/タイムスタンプ
```

**例**:
```
eBay/田中/1500/25/20260328143052
```

**内訳**:
- リサーチ方法: eBay
- 担当者: 田中
- 利益額: 1500円
- 利益率: 25%
- タイムスタンプ: 2026年3月28日 14:30:52

### なぜSKUが重要？

**問題**: 複数人が同時に出品ボタンを押すと、同じ行に書き込んでデータが上書きされる

**解決**: SKUを先に出力して行を予約する

**仕組み**:
1. 出品ボタンを押す
2. SKUを生成して出品シートの空き行に先に書き込む
3. 画像ダウンロード中（時間がかかる）も、他の人は次の行を使う
4. 上書き競合を完全に防止

### ポリシーごとの違い

各ポリシー（発送方法）で利益額・利益率が異なります。

| ポリシー | 発送方法 | 特徴 | 利益 |
|----------|----------|------|------|
| Expedited | FedEx/DHL等 | 速く届く、送料高い | 利益率低め |
| Standard | 日本郵便等 | 通常配送、送料中間 | 利益率中間 |
| 書状 | 日本郵便書状 | 安い、薄い商品のみ | 利益率高め |

リサーチシートのE14～H16行に各ポリシーの計算結果が表示されるので、それを見て最適な発送方法を選びます。

## よくある質問

### Q1. SKUが出品シートに表示されない
**A**: ヘッダー行（3行目）に「SKU」列が存在するか確認してください。

### Q2. 画像がGoogleドライブに保存されない
**A**: 以下を確認してください：
- ツール設定シートの「画像フォルダ」URLが正しいか
- そのフォルダに編集権限があるか
- メニュー > OAuth認証 を実行したか

### Q3. 仕入元が「不明」と表示される
**A**: ツール設定シートの「仕入元」「URL」列に、該当するサイトが登録されているか確認してください。

### Q4. 「アクセスが拒否されました: DriveApp」エラー
**A**: メニュー > OAuth認証 を実行して、Googleドライブへのアクセス権限を付与してください。

### Q5. 発送方法列にエラーが出る
**A**: 出品シートのCS列（発送方法）にデータ入力規則が設定されている場合、許可されている値（FedEx, DHL, 日本郵便）以外は入力できません。

### Q6. 複数人で同時に出品ボタンを押しても大丈夫？
**A**: はい、SKU先行出力により行が予約されるため、データが上書きされることはありません。

## トラブルシューティング

### 問題: 出品ボタンを押してもエラーが出る

**確認1**: eBay APIキーが正しく設定されているか
- ツール設定シートのApp ID, Cert ID, Dev IDを確認

**確認2**: OAuth認証を実行したか
- メニュー > OAuth認証 を実行
- 初回のみ必要

**確認3**: eBay URLが正しいか
- リサーチシートのItem URLにeBayの商品URLを入力しているか

### 問題: 画像が出力されない

**確認1**: 商品ページURLが正しいか
- メルカリ: `https://jp.mercari.com/item/m*****`
- ヤフオク: `https://page.auctions.yahoo.co.jp/*****`

**確認2**: 画像フォルダURLが正しいか
- Googleドライブのフォルダを開いたときのURL全体をコピーする

**確認3**: 実行ログを確認
- 表示 > ログ で詳細なエラーメッセージを確認

### 問題: 列がずれている

**原因**: 出品シートの列構造が変わった

**解決**: このツールはヘッダー名でマッピングするため、列を追加・削除しても自動で対応します。
ヘッダー行（3行目）のヘッダー名を変更しないでください。

## 運用のポイント

### 1. 複数人での作業

- ✅ **できること**: 同じリサーチシートを複数人でコピーして使う
- ✅ **できること**: 同時に出品ボタンを押す（SKUで行予約）
- ⚠️ **注意**: リサーチシートのデータは各自のコピーで管理

### 2. データのバックアップ

出品シートのデータは定期的にバックアップを取ることを推奨します。

### 3. エラーが発生したら

1. 表示 > ログ を開いて詳細エラーを確認
2. エラーメッセージをコピーして管理者に報告
3. 必要に応じてOAuth認証を再実行

## 用語集

| 用語 | 意味 |
|------|------|
| SKU | Stock Keeping Unit（在庫管理番号）。商品を一意に識別する番号 |
| Item Specifics | eBayの商品仕様（Brand, UPC, Colorなど） |
| OAuth認証 | Googleドライブなどへのアクセス権限を付与する認証 |
| ポリシー | 発送方法のこと（Expedited, Standard, 書状） |
| ヘッダー行 | 出品シートの3行目。列名が書かれている |
| 動的マッピング | 列位置が変わっても、ヘッダー名で自動的に列を見つける仕組み |
